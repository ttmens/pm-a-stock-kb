# 03 — 产品需求文档（PRD）：A股全量信息检索知识库

## 产品愿景

为个人量化投资者提供零订阅费、单VPS部署的A股知识库，以**个股事件链**为核心组织方式，支持毫秒级全文检索、LLM情感因子生成和ML就绪数据导出。

## 目标用户

| Persona | 描述 | 核心需求 |
|---------|------|----------|
| P1 个人量化投资者 | 具备Python/SQL基础，有策略回测需求 | 快速获取个股事件链和因子数据 |
| P2 独立研究员 | 深度挖掘行业/板块的政策与公告关联 | 跨股票批量检索和文档阅读 |
| P3 中小型私募操盘手 | 监控持仓股票舆情与资金异动 | 轻量工具快速响应市场变化 |
| P4 ETL调度Agent | 系统角色，无人值守运行数据管道 | 定时采集、清洗、分析、告警 |

## MVP范围

**In Scope（P0-P1）**：
- 个股事件链查询与浏览（P0）
- 全文搜索公告/研报/舆情（P0）
- 因子数据查看与CSV导出（P1）
- 系统状态监控与手动触发ETL（P1）
- 单用户Token认证（P1）

**Out of Scope（MVP阶段）**：
- 多用户/权限管理
- 盘中实时数据推送
- 策略回测引擎
- 移动端原生App
- 全市场5000+股票覆盖（MVP聚焦沪深300）

## 用户故事（≤5）

### US-1：作为个人投资者，我想按时间线查看某只股票的完整事件链，以便快速了解该股票的近期动态

**旅程映射**：J1-1 → J1-2 → J1-3 → J1-4 → J1-5（核心路径1）
**屏幕映射**：S1（搜索仪表盘）→ S2（个股事件链）→ S6（事件详情面板）

**验收标准**：
1. 输入股票代码或名称，3秒内显示匹配结果
2. 点击股票后，按时间倒序展示事件链，包含公告/财报/资金/舆情四种类型
3. 每条事件显示：标题、时间、类型标签、情感因子色块（正面绿色/负面红色/中性灰色）
4. 点击事件可展开查看详情：完整内容、来源链接、发布时间
5. 支持按事件类型过滤，过滤后即时刷新
6. 支持选择时间范围：近7天/近30天/近90天/自定义
7. 沪深300成分股事件链数据覆盖率≥80%（H4假设）

**优先级**：P0（核心差异化功能）

---

### US-2：作为研究员，我想搜索包含特定关键词的公告和研报，以便发现市场热点和关联信息

**旅程映射**：J2-1 → J2-2 → J2-3 → J2-4 → J2-5（核心路径2）
**屏幕映射**：S1（搜索仪表盘）→ S3（搜索结果页）→ S6（全文查看器）

**验收标准**：
1. 输入关键词后，P95搜索延迟<100ms（H2假设）
2. 搜索结果列表显示：标题（关键词高亮）、摘要（关键词高亮）、股票代码、发布时间、事件类型
3. 支持按股票代码、时间范围、事件类型过滤搜索结果
4. 点击结果可查看原文全文，关键词持续高亮
5. 全文查看器侧栏展示该文档关联的个股事件链摘要（≥3条相关事件）
6. 支持导出搜索结果列表为CSV/JSON格式

**优先级**：P0（核心检索能力）

---

### US-3：作为量化投资者，我想查看和导出股票的因子数据，以便直接用于机器学习模型训练

**旅程映射**：J3-1 → J3-2 → J3-3 → J3-4（核心路径3）
**屏幕映射**：S4（因子数据页）

**验收标准**：
1. 选择股票后展示因子数据表格，包含：日期、因子名称、因子值
2. 支持按因子类型筛选（情感因子/动量因子/波动率因子）
3. 支持按时间范围筛选，表格支持按日期/因子值排序
4. 提供"图表"切换按钮，可切换为折线图/柱状图展示因子时间序列
5. 点击"导出CSV"后，浏览器自动下载CSV文件，格式可直接用于ML训练（列名：stock_code, factor_date, factor_name, factor_value）
6. 情感因子由本地LLM生成，单条处理时间<3秒（H3假设）

**优先级**：P1（ML就绪核心需求）

---

### US-4：作为系统使用者，我想查看数据采集和ETL管道的运行状态，以便确认数据已更新至最新交易日

**旅程映射**：J4-1 → J4-2
**屏幕映射**：S5（系统状态页）

**验收标准**：
1. 仪表盘展示各组件健康状态：PostgreSQL、Elasticsearch、Redis、本地LLM（绿/黄/红指示灯）
2. 显示最近一次ETL运行信息：开始时间、结束时间、采集数量、成功率、耗时
3. 系统连续运行72小时无OOM事件，内存使用率<85%（H1假设）
4. 数据新鲜度标识：显示"数据已更新至 T+1 YYYY-MM-DD"或"数据落后 X 天"
5. 若ETL运行失败，显示错误摘要和最后3条错误日志

**优先级**：P1（运维可见性）

---

### US-5：作为系统使用者，我想手动触发数据采集或ETL任务，以便在数据缺失时主动补救

**旅程映射**：J4-3 → J4-4
**屏幕映射**：S5（系统状态页）

**验收标准**：
1. 提供"手动触发采集"按钮，点击后确认对话框提示影响范围
2. 提供"重新运行ETL"按钮，可选择重新处理指定日期的数据
3. 任务提交后显示进度指示器（排队中/运行中/完成/失败）
4. 任务完成后页面自动刷新，显示最新数据状态
5. 手动触发操作需Token认证（单用户简单token）

**优先级**：P1（运维操作能力）

## 非功能需求

| 指标 | 目标 | 验证方法 |
|------|------|----------|
| 全文检索P95延迟 | <100ms | 1000次随机查询基准测试 |
| 事件链查询响应 | <200ms | 300只股票各查询10次 |
| LLM情感分析吞吐 | <3秒/条 | 1000条批量处理计时 |
| 系统可用性 | 99%（月度） | 监控服务健康检查 |
| 内存使用率 | <85% | 72小时压力测试 |
| MVP数据覆盖 | 沪深300成分股≥80%数据类型 | 数据源覆盖率统计 |

## 技术约束

- 单VPS部署，建议16GB RAM（ADR-005）
- MVP阶段仅覆盖沪深300成分股（300只）（ADR-006）
- 数据新鲜度T+1，不追求盘中实时（ADR-004）
- 单用户架构，无需多租户（ADR-003）
- 本地LLM部署（Qwen2.5-7B-GGUF），零API成本（ADR-005）
