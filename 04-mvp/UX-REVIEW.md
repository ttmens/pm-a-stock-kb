## Issues 列表

### P1 — 功能缺失

| # | 问题 | 影响旅程 | 建议 |

### P2 — UI/UX 细节

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| P2-1 | 搜索结果页无搜索框复用（用户需返回首页重新搜索） | 交互效率 | 在搜索结果页顶部保留搜索框 |
| P2-2 | 因子页面无时间范围选择器（固定 30 天） | J3-2 | 添加 7/30/60 天选择器 |
| P2-3 | ETL 日志区域仅静态占位文本 | J4-2 | 调用 `/api/schedule/status` 填充真实数据 |
| P2-4 | 空状态仅用文字 "暂无数据"，未使用 DESIGN.md 的 empty state 样式 | 一致性 | 添加 `.empty-state` CSS 类（已有 emoji icon） |
| P2-5 | 搜索无 loading 状态（API 调用期间无反馈） | 即时反馈 | 搜索按钮添加 loading spinner |
| P2-6 | 无键盘导航支持（Tab 键无法遍历交互元素） | 无障碍 | 添加 focus-visible 样式 |

### P3 — 建议优化

| # | 问题 | 建议 |
|---|------|------|
| P3-1 | 前端 CSS 未使用 CSS 变量（硬编码颜色值） | 迁移到 `:root { --bg: ... }` 变量 |
| P3-2 | 所有 JS 在全局作用域 | 使用 IIFE 或模块封装 |
| P3-3 | 无错误重试机制（网络失败后需手动刷新） | 添加自动重试 |
| P3-4 | 因子图表仅支持情感因子 | 添加因子类型下拉选择 |
