# CONTEXT — A股全量信息检索知识库

## 术语表（Glossary）

| 术语 | 定义 |
|------|------|
| **Bronze层** | 数据湖三层架构中的原始层，存储不可变的原始采集数据。一旦写入即冻结，用于审计和重放。 |
| **Silver层** | 清洗后的结构化数据层，完成去重、格式统一、实体对齐，是下游消费的主要数据源。 |
| **Gold层** | 聚合/特征层，包含多维因子计算结果、ML训练特征集和归档摘要，直接服务于分析和策略。 |
| **个股事件链** | 按时间顺序串联某只股票相关的所有事件（公告、财报、资金异动、舆情），形成可追溯的因果链。 |
| **因子IC（Information Coefficient）** | 因子值与未来收益率的秩相关系数，用于评估因子的预测能力。IC > 0.03 通常认为因子有效。 |
| **T+1数据** | 交易日结束后（次日）才可用的数据，区别于盘中实时数据。本项目的数据新鲜度目标。 |
| **倒排索引** | 从关键词到文档列表的映射结构，Elasticsearch 的核心检索机制，支持全文搜索。 |
| **哈希去重** | 对采集内容计算哈希值（如SHA-256），相同哈希视为重复内容，确保数据唯一性。 |
| **ETL管道** | Extract-Transform-Load 数据流水线，负责从 Bronze→Silver→Gold 的逐层转换。 |

## 领域上下文

- **A股市场**：中国大陆沪深两市，约5000+上市公司
- **数据源类型**：财报/公告（官方披露）、资金流（交易数据）、政策（政府发布）、研报（机构产出）、舆情（社交媒体）
- **用户画像**：个人量化投资者，具备Python/SQL基础，有策略回测需求
- **部署环境**：单台VPS，Docker Compose编排

## 关键实体

- **股票**：以股票代码（如 600519.SH）为唯一标识
- **板块**：行业/概念分类，一只股票可归属多个板块
- **事件**：发生在某只股票或板块上的离散信息单元
- **因子**：从原始数据中计算得出的数值特征，用于ML和选股
