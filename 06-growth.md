# 06 — 增长策略：A股全量信息检索知识库

## 1. 北极星指标（North Star Metric）

### 业务类型判定：Productivity Game（效率游戏）

本产品是**个人量化投资者的研究效率工具**。用户价值不在于停留时长或交易次数，而在于**多快地完成个股研究并导出可用数据**。竞品 Wind/iFinD/Choice 是 SaaS 订阅模式，本项目是零订阅费、单 VPS 自部署的生产力工具。

### 北极星指标

> **周活跃研究会话数（Weekly Active Research Sessions, WARS）**
>
> 定义：单个部署实例在一周内产生的"有效研究会话"总数。一个有效研究会话 = 用户执行了至少一次事件链查询、全文搜索或因子数据导出操作（三次操作之一即计为一个会话，同一用户同一次连续操作计为一次会话）。

### 为什么选择 WARS（七项标准验证）

| 标准 | 验证 |
|------|------|
| 易于理解 | "一周内用户做了多少次研究查询"——团队每个人都懂 |
| 以客户为中心 | 衡量用户实际获得的研究价值，而非部署数量或服务器收入 |
| 可持续价值 | 周复一周的活跃会话反映习惯养成和长期留存 |
| 愿景对齐 | 产品愿景是"为个人量化投资者提供零订阅费的 A 股知识库"——用户每周都在用它做研究，就是愿景兑现 |
| 可量化 | 通过 FastAPI 日志统计查询次数，按周聚合 |
| 可行动 | 数据覆盖率、搜索速度、部署体验都直接影响会话数 |
| 领先指标 | 周活跃会话增长预示用户留存和口碑传播，先于任何变现信号 |

### 输入指标（Input Metrics）

| # | 输入指标 | 定义 | 影响 NSM 的路径 | 当前基线 | 90 天目标 |
|---|---------|------|----------------|---------|----------|
| I1 | **部署成功率** | 首次下载项目到完成 `docker compose up` 并可访问的用户比例 | 部署失败 = 零会话。降低首次使用摩擦是增长的第一道闸门 | N/A（新产品） | ≥70% |
| I2 | **沪深 300 数据覆盖率** | 沪深 300 成分股中，事件链数据≥80% 类型覆盖的个股比例 | 覆盖率越高 → 用户搜索命中率越高 → 会话越多 | 0%（MVP 待部署） | ≥80% |
| I3 | **核心查询响应 P95** | 事件链查询 <200ms + 全文搜索 P95 <100ms | 响应越快 → 用户越愿意多次查询 → 会话数增加 | N/A | 达标（见 PRD 非功能需求） |
| I4 | **因子导出转化率** | 查看因子数据的用户中，实际点击"导出 CSV"的比例 | 导出是"ML 就绪"价值的终态行为，高转化 = 用户真正获得 ML 可用数据 | N/A | ≥40% |
| I5 | **7 日留存率** | 首次完成研究会话的用户中，7 天内再次发起会话的比例 | 留存直接乘入周活跃会话基数 | N/A | ≥35% |

### 指标层级关系

```
WARS（北极星指标）
├── I1 部署成功率 ──→ 激活用户基数
├── I2 数据覆盖率 ──→ 查询命中率 ──→ 每次会话的查询次数
├── I3 查询响应速度 ──→ 用户体验 ──→ 单次会话持续时长
├── I4 因子导出转化率 ──→ 价值感知 ──→ 7 日留存
└── I5 7 日留存率 ──→ 回访用户数 ──→ 周活跃会话数
```

## 2. 目标客户画像（ICP）

### 主要 ICP：个人量化投资者（P1）

| 维度 | 描述 |
|------|------|
| **角色** | 个人投资者 / 独立量化研究员 |
| **技术能力** | 会使用 Python/SQL，有策略回验经验 |
| **现有工具** | AKShare/Tushare（数据接口）+ Wind/iFinD（试用版或过期订阅）+ 自建脚本 |
| **痛点** | 机构终端年费 1.4 万~10 万+，个人负担不起；开源工具只有数据接口，缺少知识库和全文检索；信息散落在雪球/淘股吧，缺乏事件链组织 |
| **核心需求** | 快速获取个股事件链和因子数据，直接用于 ML 模型训练 |
| **购买/采用决策** | 自部署，决策者是使用者本人，无采购流程 |
| **触达渠道** | 知乎量化话题、CSDN 技术博客、JoinQuant/RiceQuant 社区、GitHub 开源项目 |

### 次要 ICP

| Persona | 优先级 | 特征 | 触达策略 |
|---------|--------|------|---------|
| **P2 独立研究员** | 中 | 深度挖掘行业/板块政策与公告关联，需要跨股票批量检索 | 技术博客 + 案例研究（"用事件链分析 XX 行业政策传导"） |
| **P3 小型私募操盘手** | 低 | 监控持仓舆情与资金异动，需要轻量快速响应 | 产品猎快 + 私募社区口碑 |
| **P4 ETL 调度 Agent** | 内部 | 系统角色，无人值守数据管道 | 内部文档 + API 文档 |

## 3. 核心价值主张与消息传递

### 核心价值主张

> **零订阅费的 A 股事件链知识库——个人量化投资者的研究加速器**
>
> 不用每年花 3.98 万买 Wind。在自己的 VPS 上部署一个知识库，以个股事件链为核心组织方式，毫秒级全文检索，LLM 情感因子分析，CSV 导出直接用于机器学习。

### 差异化定位矩阵

| 维度 | Wind/iFinD/Choice | AKShare/Tushare | 本项目 |
|------|-------------------|-----------------|--------|
| 价格 | ¥1.4 万~10 万+/年 | 免费（仅接口） | **零订阅费** |
| 数据组织 | 扁平数据库 | 无知识库 | **事件链时间线** |
| 检索能力 | 条件筛选 | 无 | **毫秒级全文检索** |
| AI 分析 | 问答/摘要 | 无 | **LLM 情感因子 + ML 就绪导出** |
| 部署 | SaaS | 本地脚本 | **单 VPS Docker Compose** |
| 数据架构 | 不透明 | 无分层 | **Bronze→Silver→Gold 三层** |

### 渠道消息变体

| 渠道 | 消息角度 | 示例标题 |
|------|---------|---------|
| **知乎/技术博客** | 技术深度 + 性价比对比 | "我自建了一个 A 股知识库，替代了每年 3.98 万的 Wind" |
| **GitHub** | 开源 + 可复现 | 项目 README 强调架构清晰、一键部署、数据分层 |
| **量化社区** | 实战案例 + 因子产出 | "用事件链做舆情因子——从采集到 ML 训练的完整流程" |
| **V2EX/独立开发者** | 个人工具 + 技术栈 | "16GB VPS 跑 PostgreSQL + ES + LLM 的 A 股知识图谱" |

## 7. 风险与缓解

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| **部署门槛过高**：目标用户有 Python/SQL 基础，但 Docker + ES + PG + LLM 的部署复杂度可能劝退 | 部署成功率低 → 零 WARS | E6 优先优化；提供预配置 VPS 镜像或一键脚本 |
| **数据合规风险**：AKShare/Tushare 等数据源的许可条款可能限制商业使用 | 项目可能面临法律风险 | README 明确"个人研究用途"；仅使用合法 API |
| **社区反响冷淡**：量化投资者可能不信任自部署方案的数据质量 | 增长实验全部失败 | 先在小范围私募群内测，获取种子用户反馈后再公开发布 |
| **VPS 成本感知**：虽然零订阅费，但 VPS 月费（¥100-200/月）可能成为障碍 | 个人用户放弃部署 | 在文章中对比"VPS 月费 ¥150 vs Wind 年费 ¥3.98 万"，凸显性价比 |
